# Rick and Morty: Clean Architecture App

Современное мобильное приложение на Flutter для просмотра персонажей вселенной "Рик и Морти". Построено на принципах чистой архитектуры, поддерживает работу без интернета и продвинутую систему поиска.

---

## 🛠 Стек технологий

* **Архитектура:** Clean Architecture (Data, Domain, Presentation) — строгое разделение ответственности.
* **State Management:** `flutter_bloc` — предсказуемое управление состоянием.
* **Локальная БД:** `Hive CE` — кэширование данных для офлайн-режима.
* **Сеть:** `Dio` — клиент для работы с REST API.
* **Ресурсы:** `flutter_launcher_icons` — автоматизированная генерация иконок под разные платформы.
* **Генерация кода:** `Freezed` и `JsonSerializable` для создания надежных моделей данных.
* **Анимации:** `Hero`, `CustomClipper`, `Implicit Animations`.
* **UX компоненты:** `Shimmer` (скелетная загрузка), `Overlay API` (кастомные уведомления).
* **Темизация:** `ThemeData` + `ThemeCubit` — бесшовная поддержка темных и светлых цветовых схем.
---

## 🚀 Реализованные возможности

### 1. Продвинутая фильтрация и поиск
* **Multi-Criteria:** Комбинируйте поиск по имени с фильтрами по статусу и полу персонажа.
* **Type-Safe Logic:** Использование `Enum` в Domain-слое для надежной обработки состояний.
* **Smooth UX:** Плавная анимация раскрытия панели фильтров и оптимизированный ввод с дебаунсом.

### 2. Главный экран и пагинация
* **Infinite Scroll:** Автоматическая подгрузка новых персонажей при прокрутке.
* **Character Cards:** Детальная информация (аватар, статус, вид, последняя локация).
* **Interactive Favorites:** Мгновенное добавление в избранное прямо из списка.
* **Micro-interactions:** Плавные анимации Implicit Animations при добавлении в избранное (изменение масштаба и цвета иконки), повышающие отзывчивость интерфейса.

### 3. Экран «Избранное»
* **Offline Access:** Просмотр списка избранных персонажей даже без подключения к сети.
* **Swipe to Delete:** Быстрое удаление персонажей из списка с помощью жеста `Dismissible`.

### 4. Работа с данными и Офлайн-режим
* **Smart Caching:** Все загруженные данные сохраняются в `Hive`. При потере связи приложение автоматически переходит на работу с локальным кэшем.
* **Persistence:** Весь прогресс и список избранного сохраняются между запусками приложения.

### 5. Продвинутый UX и Анимации
* **Shimmer Loading:** Вместо стандартных лоадеров используются скелетные заглушки, имитирующие структуру карточек.
* **Custom Hero Morphing:** Бесшовный переход аватара персонажа из списка на экран деталей с трансформацией формы из квадрата в круг через `CustomClipper`.
* **Global Toast System:** Кастомная система уведомлений через `Overlay`, позволяющая выводить сообщения поверх интерфейса без блокировки UI.

### 6. Динамическая смена тем (Light/Dark Mode)
* **Adaptive Design:** Полная поддержка светлой и темной тем. Интерфейс автоматически перекрашивается (фон, карточки, текстовые стили) при смене темы.
* **Architecture-First Theming:** Цвета не "зашиты" в виджеты, а управляются через `Theme.of(context)`, что позволяет легко добавлять новые цветовые палитры.
* **Reactive Switching:** Мгновенное переключение между режимами с помощью `ThemeCubit` без потери состояния экрана.

### 7. Кастомизация и брендинг
* **Custom App Icon:** Разработана уникальная иконка приложения в стилистике вселенной Рика и Морти.
* **Multi-platform Icons:** Настроена автоматическая генерация иконок для всех разрешений Android и iOS через `flutter_launcher_icons`, что гарантирует корректное отображение на любых устройствах.
* **Native Splash Screen:** Реализован адаптивный экран загрузки. Цвет фона и логотип автоматически подстраиваются под системную тему устройства (Light/Dark Mode) еще до полной инициализации Flutter.

## 🏗 Архитектура и качество кода

* **Разделение ответственности:** Каждый слой (Data, Domain, UI) автономен и выполняет только свою задачу.
* **Dependency Injection:** Использование внедрения зависимостей через конструкторы и сервис-локатор (`GetIt`) для гибкости кода.
* **Clean Code:** Соблюдение официальных рекомендаций Dart Lints и использование иммутабельных объектов.
* **Decoupled Styling:** Стили текста (`AppTextStyles`) полностью отделены от цветовых решений, что соответствует принципу единственной ответственности (SRP).

## ⚡ Производительность и UX

* **Smooth Scrolling:** Оптимизированные списки для плавной работы на любых устройствах.
* **Reliable Pagination:** Грамотная обработка страниц API без дублирования данных.
* **Non-blocking UI:** Все тяжелые операции (сеть, БД) выполняются асинхронно, обеспечивая стабильные 60 FPS.

## 📦 Как запустить проект

1. **Склонируйте репозиторий:**
`
git clone [https://github.com/nikitasalnikov56/rick_and_morty.git](https://github.com/nikitasalnikov56/rick_and_morty.git)`

2. **Установите зависимости:**

`flutter pub get`

3. **Сгенерируйте необходимые файлы: > ВАЖНО: Проект использует генерацию кода (Freezed, Hive, AutoRoute). Перед запуском обязательно выполните команду ниже, чтобы создать недостающие файлы:**

`dart run build_runner build --delete-conflicting-outputs`

4. **Запустите приложение:**
`flutter run`
